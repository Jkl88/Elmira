# Elmira — электронный замок

Проект на **PlatformIO** (Arduino, ESP32-C3): электронный замок с веб-интерфейсом на [TKWifiManager](https://github.com/Jkl88/TKWifiManager), открытие по паролю или магнитному ключу (AS5600).

## Оборудование

- **ESP32-C3** SuperMini (или аналог)
- Серво 360° SG90
- Датчики холла SS41F × 2 (открыто / закрыто)
- Магнитный энкодер **AS5600** (I2C) — открытие «ключом с магнитом»

## Функции

- Веб-интерфейс: 8 полей пароля, кнопки «Открыть»/«Закрыть», «Сменить пароль», «Аварийное закрытие»
- Пароль 8 цифр в NVS, по умолчанию `12341234`
- Открытие: по паролю из веба или поворотом магнита на AS5600 на 90° по часовой
- Закрытие: только из веба (GET `/close` без пароля)
- Индикация состояния по WebSocket: открыто / закрыто / в процессе / неизвестно
- API: `GET /open?pass=12345678`, `GET /close`, `GET /status` (JSON: state, hall_open, hall_closed, as5600_angle, calib)
- **Калибровка через веб:** AS5600 — «Запомнить как старт»/«как конец» (NVS); холлы — «Сохранить как открыто»/«закрыто»; режим калибровки — серва управляется поворотом магнитного ключа, в интерфейсе отображаются значения холлов.
- **Звуковая обратная связь:** пищалка на пине `PIN_BUZZER` (успех — один короткий, ошибка — два).

Подробное ТЗ и ответы на уточнения — в [.apprules](.apprules).

## Сборка и прошивка

```bash
# Сборка
pio run

# Прошивка
pio run -t upload

# Загрузка веб-страниц в LittleFS (обязательно перед первым запуском)
pio run -t uploadfs
```

После загрузки: при отсутствии сохранённых Wi‑Fi сетей поднимается AP `Elmira-XXXXXX`. Откройте http://192.168.4.1 для настройки Wi‑Fi и управления замком.

## Конфигурация

Пины, таймауты и пароль по умолчанию задаются в [src/config.h](src/config.h):

- `PIN_SERVO`, `PIN_HALL_OPEN`, `PIN_HALL_CLOSED` — пины сервы и датчиков холла
- `PIN_SDA`, `PIN_SCL` — I2C для AS5600
- `PIN_BUZZER` — пищалка (0 = отключена)
- `SERVO_DIR_OPEN`, `SERVO_DIR_CLOSE` — направление вращения
- `SERVO_TIMEOUT_MS` — таймаут сервы (ошибка, если датчик не сработал)
- `AS5600_ANGLE_START_DEG`, `AS5600_ANGLE_END_DEG` — углы для магнитного ключа (0° и 90°)
- `LOCK_DEFAULT_PASSWORD` — пароль при первом запуске
- **Энергосбережение** (при питании от батареи): `POWER_SAVE_ENABLE`, `POWER_SAVE_IDLE_DELAY_MS`, `POWER_SAVE_IDLE_CPU_MS`, `POWER_SAVE_IDLE_LOOP_MS` — см. раздел ниже.

## Энергосбережение (батарея)

Чтобы **GET, веб и магнитный ключ** работали в любой момент, устройство не переводится в deep/light sleep (иначе WiFi отключится и запросы не дойдут). Реализовано следующее:

- **WiFi** — в режиме STA используется встроенный modem sleep (драйвер сам усыпляет радио между DTIM), соединение и приём запросов сохраняются.
- **Активность** — любое обращение по HTTP/WebSocket и движение замка сбрасывает таймер простоя.
- **При простое 10+ с** — в `loop()` добавляется задержка `POWER_SAVE_IDLE_LOOP_MS` (50 мс), чтобы не крутить CPU впустую.
- **При простое 60+ с** — частота CPU понижается до 80 МГц; при следующей активности снова 160 МГц.
- **Bluetooth** — явно отключён вызовом `esp_bt_controller_disable()` (экономия ~30–80 мА).

Параметры в `config.h`: `POWER_SAVE_ENABLE` (1/0), `POWER_SAVE_IDLE_DELAY_MS`, `POWER_SAVE_IDLE_CPU_MS`, `POWER_SAVE_IDLE_LOOP_MS`. Отключить экономию: `POWER_SAVE_ENABLE 0`.

## Структура проекта

```
src/
  main.cpp           — setup/loop, TKWifiManager, магнитный ключ
  config.h           — пины и параметры
  lock_state.*       — состояния замка, открытие/закрытие/аварийное закрытие
  lock_password.*    — NVS, проверка и смена пароля
  lock_web.*         — маршруты /open, /close, WebSocket
  power_save.*       — энергосбережение (активность, CPU 80 МГц, задержка при простое)
  hardware/
    hall_sensors.*   — датчики холла SS41F
    servo_driver.*   — серва 360°
    as5600_driver.*  — AS5600, магнитный ключ
data/
  index.html         — главная страница замка (подмена в TKWifiManager)
  theme.css, theme.js — темы как в TKWifiManager
```
